The Insight

Autoregressive is not the only way

GPT commits to tokens left-to-right, never looking back. BERT sees everything at once. What if we used that power to generate?

GPT-style (Autoregressive)

Tokens appear one at a time, left to right. Each token is a point of no return.

Theanswerisdefinitely42,yes
→ sequential, committed, no revision

ChatBERT-IMR (Bidirectional)

All positions visible at once. Fills in the most confident tokens first, iteratively.

t=0[M][M][M][M][M][M]
t=1[M][M]is[M][M][M]
t=2The[M]is[M][M]yes
t=3Theansweris42,[M]yes
→ parallel, deliberative, revisable
Four Approaches

One question, four architectures

We explored four distinct ways to make BERT generate conversational text.

ED

ChatBERT-ED

Encoder-Decoder

BERT encodes the conversation context bidirectionally. A lightweight GPT-style decoder generates the response, attending to BERT's rich representations.

IMR

ChatBERT-IMR

Iterative Mask-Refinement

Pure BERT generation. Start with all [MASK] tokens, iteratively unmask the most confident predictions. Discrete diffusion meets masked language modeling.

RA

ChatBERT-RA

Retrieval-Augmented

BERT scores and ranks candidate responses from a retrieval corpus. Bidirectional understanding as a discriminator, not a generator.

FIB

ChatBERT-FIB

Fill-in-the-Blank

Template-guided generation. Given a sentence structure with blanks, BERT fills them in using full bidirectional context.

Model overview

Input: "What is the meaning of life?" | +-------------------------------+ | BERT Encoder (6L, 768H) | | Bidirectional self-attention | | Full context at every layer | +---------------+---------------+ | +---------------+---------------+ | MLM Head (project + softmax) | | Predict token at each [MASK] | +---------------+---------------+ | +-----+-----+ | Confidence | | Ranking | +-----+-----+ | Unmask top-k -> Iterate T times | Output: "The answer is 42, yes"
